Mobile is everywhere

The Problem is Choice

What are my options?

  • Mobile Web Application / Mobile Site
  • Native Application
  • Hybrid Application

Mobile Web Application / Mobile Site

Overview

  • Like bbc.co.uk and bbc.co.uk/mobile
  • Different Layout or Content according to the browser
  • Please use Responsive Design

Pros

  • Easy to build
  • No new skills required
  • Best for constantly updated content
  • No need to go to App Store
  • Host like a normal website

Cons

  • Little or no offline capability
  • Not fast / Depends on server traffic
  • Doesn't create the best of impressions

Native Application

Overview

  • Built with Objective-C for iOS, Java for Android etc.
  • Different codebases for different devices

Pros

  • Best Performance
  • Great for graphics
  • Will work offline (ofcourse)
  • Native Look and Feel
  • Leverage the full platform API

Cons

  • New languages / API to learn
  • Larger Teams / Longer Shipping time
  • Multiple codebases to maintain
  • App Store Submission and Approval (on iOS)

Hybrid Application

Overview

  • Built with a framework
  • Works on most popular devices with a single codebase

Pros

  • Quick Development
  • Can use some device hardware capabilities (depends on framework)
  • Decent performance
  • Customizable Look and Feel

Cons

  • Doesn't perform as well as Native apps
  • Good for form based / simple UI apps
  • Dependency on frameworks and their stability

Titanium Appcelerator

  • Code in JavaScript
  • Use Titanium's API (rapidly evolving)
  • Generates native UI components
  • Good performance
  • Add-On libraries available

Corona

  • Code in Lua, a light fast language
  • Good performance
  • Even works for game development
  • Add-On libraries available

Rhomobile

  • Code in Ruby on a Rails like framework
  • Backed by Motorola
  • Quick development cycle
  • Good for enterprise / Client-Server synchronization
  • API undergoing transition

RubyMotion

  • Code in Ruby
  • iOS Wrapper API in Ruby
  • Good Performance as it generates native code
  • Only for iOS
  • Still need to learn iOS API

Phonegap

  • Code in HTML5 + JavaScript
  • Super quick development
  • App runs on WebView
  • Platform API available in JavaScript
  • (Maybe) An uncertain future
  • Can be slow

Sencha Touch / jQuery Mobile

  • Code in HTML5 + JavaScript
  • Provide native look and feel
  • Apps runs on WebView / Webkit
  • APIs under rapid development
  • Have to be paired with Phonegap for native deployment
  • Can be slow

Wait there is more ...

  • KendoUI
  • Dojo
  • Haxe
  • More frameworks being born every week

Choose wisely

  • Ask yourself what do you want
    • Performance
    • Great graphics / Gaming
    • Ease of development
    • Low cost
    • What is your feature set
